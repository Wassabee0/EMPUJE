Empuje convierte recursos ocultos en oportunidades revisadas.

Una web pública para explicar el concepto, no para listar oportunidades reales. Las ofertas, necesidades, evidencias y matches viven dentro de cuentas privadas después de revisar cada claim.

Regla central

Lo público enseña ejemplos. Lo real se revisa.

Una oferta no se muestra a otros miembros hasta tener contexto, evidencia y capacidad.

Contenido que falta en una landing genérica.

La landing tiene que explicar la tesis, enseñar ejemplos suficientes y aclarar el modelo de confianza sin enseñar datos privados de miembros.

Problema

La gente tiene recursos, pero no sabe convertirlos en colaboraciones.

Espacios, stock, canales, experiencia, contactos o tiempo suelen estar infrautilizados.

Cómo funciona

Aplicar, revisar, verificar y aprobar matches privados.

Empuje prioriza coordinación y confianza por encima de volumen o listados abiertos.

Trust model

Ningún claim se trata como verdad hasta revisarlo.

La beta debe ser explícita: oferta pendiente, evidencia faltante o capacidad agotada bloquean intros.

Onboarding

Aplicar no es rellenar un formulario vago.

El usuario debe saber exactamente qué necesita para que su oferta pase una revisión inicial.
