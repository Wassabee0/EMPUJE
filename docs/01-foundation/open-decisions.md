# Decisiones

Estado: activo, 2026-05-26.

## Cerradas Para Esta Beta

| Decisión | Elección |
| --- | --- |
| Despliegue | Next.js App Router desplegable en Vercel. |
| Persistencia | Supabase Auth, Postgres y Storage. |
| Auth | Supabase Auth como backend; Google como proveedor OAuth dentro de Supabase. |
| Signup | Crea cuenta Supabase y perfil pendiente de revisión. |
| Verificación | La evidencia se revisa por oferta antes de confiar en una afirmación. |
| Visibilidad pública | La web pública muestra ejemplos; oportunidades reales son privadas. |
| Matching | Determinista primero; admin puede aprobar o ajustar manualmente. |
| Monetización | Cobrar por confianza, coordinación, acceso curado y círculos, no por estar listado. |

## Abiertas Antes de Producción Pública

| Decisión | Por qué importa |
| --- | --- |
| Política de privacidad | El signup puede incluir evidencia de negocio y datos de contacto. |
| Aviso legal/fiscal | Las colaboraciones pueden tener implicaciones fiscales; no prometer trueque informal sin límites. |
| Primera ciudad o vertical | La liquidez será más fuerte si el primer grupo es estrecho. |
| Email admin inicial | El fundador debe crear/promocionar su cuenta tras configurar Supabase. |
| Mensajes transaccionales | Harán falta emails para evidencia solicitada, aprobación e intros. |
